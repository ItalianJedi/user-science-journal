Reflections on Accessbility and Analytics

Posted by Tony Graf on December 3, 2018

Last week we set up goals. My goal was to see if the user will watch a video on web accessibility for 5 minutes. I thought it was an interesting video on an accessibility tool called Lighthouse that was added to Google Chrome. According to Google Analytics, two users completed that goal.

This week we are to set up content experiments on our site. Content experiments are slight changes intended to have more people complete the goal. I tried to set up one experiment but was unable to get it set up correctly. Then our instructor showed us a different experiment to set up which seemed to work a lot better for me.

Experiment 1: I made a new post with a different video than the Lighthouse one. This was also from the public archives and dealt with Web Accessibility. Accessibility Matters: Creating a Better Web

I didn’t start this experiment due to an error I came across.

It looks like I was able to complete the experiment but my instructor gave us a different experiment to run instead.

Experiment 2: Google Optimize! Google Optimize is a new A/B test that Google has created. It will eventually replace the old way of doing content experiments so it is important that we are used to doing these types of testing.

With this experiment, instead of using the Accessibility Matters video, the variant I set up was just a slight change in the title of the post. Instead of “What’s New in Web Accessibility (Google I/O ’18)” I just used “What’s New in Web Accessibility.” Other than sending the user to another video, I wasn’t quite sure how to set up an experiment that would be really useful for analysis. Luckily, I believe the goal of these projects is to demonstrate that we are able to set up these content experiments.

The duration of the goal that I set up was three days and it looks like it had two experiment sessions, which was more than I expected.
